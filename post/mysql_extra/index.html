MySQL 高级用法

分页

  • 分页查询是数据库的方言，不同的数据库有不同的实现。例如，MySQL 中是 LIMIT，oracle 中是 ROWNUM，sql server 中是 ROW_NUMBER。

用法：

SELECT  FROM 【表名】 LIMIT 【分页起始行】,【分页大小】
  • 起始索引从 0 开始，起始索引 = （查询页码 - 1）* 每页显示记录数。
  • 在 MySQL 中，如果分页查询的是第一页数据，起始索引可以省略，直接简写为 LIMIT 【记录数】。

约束

保证数据的正确性、完整性、有效性

  • 主键约束
  • 唯一约束
  • 非空约束
  • 检查约束
  • 外键约束
  • 默认值约束

主键约束

  • 特点：
    • 唯一：一张表只能有一个主键约束
    • 非空：主键不能为空
  • 关键字：PRIMARY KEY
  • 设置方法：
    • 在列的后面声明
    • 在创建表的时候，在所有列声明之后，单独起一行声明
    • 修改表的时候进行声明
  • 设置建议
    • 主键不要设置在有意义的列
    • 修改数据时不要修改主键

唯一约束

  • 特点：值唯一，允许出现一个空值
  • 关键字：UNIQUE

外键约束

  • 外键：外键是在一张表中引用的另一张表的主键。
  • 外键约束：用于在两个表之间建立关系，限制外键只能是引用指定表中指定列的值
  • 在互联网开发中，外键一定存在，但是外键约束一般不加

数据库的备份与恢复

备份

  • 命令行：mysqldump -u 【用户名】 -p【密码】 【数据库名】 > 【sql文件路径】

    使用 mysqldump 导出的 SQL 文件只有数据表和其中数据，并不包含数据库

恢复

  1. SOURCE 【sql 文件路径】
  2. mysqldump -u 【用户名】 -p【密码】 【数据库名】 < 【sql文件路径】

数据库的多表查询

  • 方式：
    • 联合查询
    • 表连接查询
    • 子查询

联合查询

把多个 SELECT 查询的结果连接起来

关键字：

  • UNION（去重）
  • UNION ALL（不去重）

要求：结果集的列的个数相同，列含义相同

连接查询

关键字：JOIN

方式：

  • 内连接
  • 外连接
    • 左外连接
    • 右外连接
  • 自然连接（列出两个表的笛卡尔积，不常用）
  • 自连接

写法：

  • 标准写法
SELECT * FROM 【表A】 JOIN 【表B】;
  • 非标准写法
SELECT * FROM 【表A】 , 【表B】;
  • 如果多张表有同名列, 使用 表名(表别名).列名 区分
SELECT * FROM 【表A】 JOIN 【表B】
ON 【表A】.【列名】 = 【表B】.【列名】;

n 张表连接，需要最小 n-1 个表相连
